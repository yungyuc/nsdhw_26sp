Problem to Solve
================

Semiconductor electrical characterization, such as data generated by 
the **B1500** measurement system, often results in large CSV files 
with dynamic schemas depending on the specific test recipes 
(e.g., IV, CV, or breakdown voltage tests). 

Processing these varied datasets using traditional row-based structures 
is inefficient for statistical analysis across thousands of samples. 
This project implements a **Columnar Array (Struct of Arrays, SoA)** 
library to provide:

1. Efficient memory management for high-throughput semiconductor data.
2. Flexibility to handle dynamic columns corresponding to different 
   measurement parameters.
3. High-performance numerical filtering in C++ before passing data to 
   Python for visualization.

System Architecture
===================

The system will be a hybrid C++/Python library:

1. **C++ Core**: 
   * A template-based ``ColumnarArray`` class to manage contiguous memory 
     buffers for each measurement parameter.
   * Efficient iterators and row-accessors to handle data filtering and 
     basic statistics.
2. **Python Wrapper**: 
   * Using **pybind11** to expose the C++ data structures to Python with 
     zero-copy overhead.
3. **Data Ingestion**: 
   * A Python-based parser to handle the irregular headers and formats of 
     B1500 CSV files, populating the underlying C++ columnar storage.

API Description
===============

Example usage of the proposed library:

.. code-block:: python

    import semi_data_engine as sd

    # Load B1500 CSV data into high-performance columnar storage
    data = sd.load_csv("measurement_b1500_recipe_A.csv")

    # Access a specific column (e.g., Drain Current 'Id')
    # The calculation is performed in C++ for maximum speed
    mean_id = data.column("Id").mean()

    # Pass the column directly to Matplotlib or Pandas without copying data
    data.plot_iv_curve(x="Vd", y="Id")
