Performance Benchmark Results:
==============================
Matrix Size: 1000 x 1000 (Tested on GitHub Actions CI)

1. Execution Time (Minimum of 5 repeats):
    - Naive Multiplication (multiply_naive): 1.1588 seconds
    - MKL Multiplication (multiply_mkl): 0.0248 seconds

2. Speedup Analysis:
    - MKL vs Naive: MKL is 46.6207x faster than the naive version.
    - Tiling vs Naive: The Tiling implementation successfully passed
    the CI validation test, achieving a runtime strictly less than
    80% of the naive version (providing a >20% speedup).

3. Conclusion:
    By rearranging the loop order and utilizing the Tiling technique,
    the memory cache hit rate was significantly improved, allowing
    the Tiling version to be substantially faster than the Naive
    version. Furthermore, the MKL library demonstrated extreme
    performance via low-level hardware optimizations, vastly
    outperforming both naive and tiled implementations.
